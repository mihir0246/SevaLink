# SevaLink: Revolutionizing Volunteer Management

## The Problem We Solved
Non-Governmental Organizations (NGOs) and community groups often struggle with an overwhelming administrative burden: matching the right volunteers to the right tasks. Traditional systems rely on manual sorting, resulting in wasted time, delayed impact, and frustrated volunteers whose unique skills go unutilized. 

**SevaLink** solves this by intelligently bridging the gap between NGOs and volunteers. Our platform leverages artificial intelligence to automate the matchmaking process, ensuring that every volunteer is recommended for tasks that perfectly align with their skills, location, and availability.

---

## What Makes SevaLink Unique?
1. **AI-Powered Matchmaking:** We integrated **Google Gemini 1.5 Flash** to act as a digital recruiter. It automatically analyzes volunteer profiles against incoming NGO requests and generates a "Match Grade," instantly recommending the best candidates.
2. **Automated Geocoding Engine:** Volunteers don't need to enter rigid map coordinates. They simply provide a city name (like "Dombivli"), and our backend uses the **OpenStreetMap Nominatim API** to dynamically resolve it into exact geospatial coordinates for geographic tracking.
3. **Role-Based Workflows:** A highly secure dual-dashboard system. Administrators and NGOs receive a command center to view AI-matched candidates, while Volunteers get a gamified, beautiful dashboard showing their impact, hours contributed, and matched tasks.
4. **Resilient Production Deployment:** Engineered to run smoothly across cloud boundaries, utilizing secure cross-origin resource sharing (CORS) and a custom proxy pipeline to securely bridge AWS and Render environments without Mixed Content errors.

## 🛠 Our Technology Stack

### ⚡ Infrastructure & Deployment
*   **AWS Amplify:** High-speed, secure global CDN for our React frontend with automated continuous deployment.
*   **Render:** High-performance Backend-as-a-Service hosting our Node.js runtime with free native SSL routing.
*   **Google Cloud Console:** Enterprise-grade OAuth 2.0 Identity tracking.

### 🧠 Core Architecture (MERN Stack)
*   **MongoDB Atlas:** Global cloud database hosting our NoSQL cluster for incredibly fast, flexible data read/writes.
*   **Express.js & Node.js:** The core API engine handling secure routing, authorization, and data extraction.
*   **React 18 & Vite:** Lightning-fast frontend providing a fluid, Single-Page Application (SPA) experience.
*   **TailwindCSS & Framer Motion:** Beautiful, responsive utility-first styling paired with modern micro-animations that make the interface feel alive and dynamic.

### 🤖 Intelligence & APIs
*   **@google/generative-ai (Gemini 1.5 Flash):** The brain behind our matchmaking service, converting unstructured volunteer data into structured, graded assignments.
*   **node-geocoder (OpenStreetMap):** Intelligent location parsing.
*   **Passport.js:** Streamlined, secure single sign-on (SSO) bridging our MongoDB users with Google's authentication framework.

---

## System Architecture Flow
1. **Authentication:** User signs in via Google OAuth → Token generated by Passport → Token saved securely in local storage.
2. **Data Intake:** Volunteer fills out skills and location → Node.js intercepts the location → Nominatim API converts it to Lat/Long → Profile saved to MongoDB.
3. **AI Assignment:** NGO creates a task → Express API grabs the top volunteers → Payload sent to Gemini AI → Gemini grades the matches → Results streamed instantly to the NGO's React Dashboard.
